# wmnet 汉化 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add full Chinese/English bilingual support to wmnet with ~90 translated strings and runtime language switching.

**Architecture:** New `src/i18n.ts` module holds translation tables for en/zh. `t(key, lang?)` lookup function used everywhere UI strings are output. `app.ts` handles `l` key for runtime toggle. `types.ts` Settings schema extended with `language` field. `index.ts` CLI commands use t() for all descriptions.

**Tech Stack:** TypeScript, Bun

export function t(key: string, lang?: Lang): string {
  const l = lang ?? currentLang;
  return translations[l]?.[key] ?? translations.en?.[key] ?? key;
}

export function setLang(lang: Lang): void {
  currentLang = lang;
}

export function getLang(): Lang {
  return currentLang;
}
