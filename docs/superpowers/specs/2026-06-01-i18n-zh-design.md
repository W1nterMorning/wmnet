# wmnet 汉化设计规格

## 目标

为 wmnet CLI 工具添加完整的中英文双语支持，约 50 个 UI 字符串全覆盖。

## 语言切换方式

- **持久化默认**：`settings.language` 写入 `~/.config/wmnet/profiles.yaml`，CLI `wmnet settings --lang zh` 修改
- **运行时切换**：交互模式按 `l` 即时切换，不写配置，下次启动恢复默认

## 术语翻译格式

中文 + 括号英文缩写：`网关 (Gateway)`、`动态主机配置 (DHCP)`、`域名系统 (DNS)`、`静态 IP (Static IP)`

## 架构

新增文件 `src/i18n.ts`，修改 `src/types.ts` `src/app.ts` `src/index.ts`：

```
src/i18n.ts    — 翻译表（en/zh）+ t(key, lang?) 函数
src/types.ts   — Settings 加 language: "en" | "zh"
src/app.ts     — 所有硬编码字符串 → t('key')
src/index.ts   — CLI 帮助 + 状态命令 → t('key')
```

## 实现策略

1. 新建 `src/i18n.ts`，导出 `t(key)` 和 `setLang(lang)`、`getLang()`
2. `types.ts` Settings 加 `language: "en" | "zh" = "en"`
3. `app.ts`：所有字符串替换为 `t('key')`，`l` 键切换语言并重渲染
4. `index.ts`：Commander .description() 和 .option() 使用 t()，settings 命令加 `--lang`
5. 中文版由于汉字宽度不同，`.padEnd()` 对齐需改用视觉宽度替代字符数，但可选不做（暂用足够大的固定宽度）
6. 字数会翻倍（英文 + 中文各一份），约 120 条字符串

## 风险

- **中文字符宽度**：`padEnd(n)` 按字符数补空格，中文 1 字 = 2 英文宽，对齐会轻微偏移。影响较小，可后续优化
- **编译后字体**：编译版无 figlet 字体，fallback ASCII logo 为纯英文，不受翻译影响

## 不在范围内

- 主题名保持英文（品牌色名）
- 配置文件名 `~/.config/wmnet/profiles.yaml` 不变
- Figlet 字体 logo 不变
- GitHub README 不翻译
